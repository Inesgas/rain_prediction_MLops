# Changelog 24.06.26
Fix UID-mismatch permission errors blocking Airflow daily ingestion DAG

The Airflow container runs as UID 50000, but host-created bind-mounted
directories (airflow/logs, data/incoming, data/raw, reports/versioning,
.dvc) were owned by the host user, causing PermissionError failures in
fetch_and_upsert_daily_weather and version_raw_weather_data tasks.
Fixed by chowning these paths to 50000:0 for container writes.

Also trims DEFAULT_LOCAL_INPUTS in dvc_versioning.py to drop legacy
preprocessed CSVs that train_winner.py no longer reads directly,
removing a false verify-local blocker.

Verified daily_weather_ingestion DAG runs end-to-end: Open-Meteo
ingestion → dvc add → freshness check (fresh, 1-day lag) → snapshot →
dvc status recording → local-only no-push notice.

mkdir -p airflow/logs airflow/dags
sudo chown -R 50000:0 airflow/logs



mkdir -p data/incoming
sudo chown -R 50000:0 data/incoming

sudo chown -R 50000:0 .dvc

## MLflow Integration: Tracking, Model Registry, DagsHub, Airflow

This PR adds end-to-end MLflow experiment tracking and model registry
integration to the rain prediction pipeline, backed by DagsHub as the
combined MLflow tracking server and DVC remote.

### What's included

**Experiment Tracking**
- Added `mlflow-skinny==3.14.0` to `requirements.txt` (lightweight client,
  avoids the `pandas<3` dependency conflict of full `mlflow`)
- Configured `MLFLOW_TRACKING_URI` / `MLFLOW_TRACKING_USERNAME` /
  `MLFLOW_TRACKING_PASSWORD` via `.env` (see `.env.example`)

**Model Registry**
- `src/models/train_winner.py`: winner CatBoost model now logs params,
  metrics, threshold, and the model artifact via
  `mlflow.catboost.log_model(..., registered_model_name="rain_prediction_catboost")`
- Verified: model registered and versioned in the DagsHub Model Registry

**Model Comparison**
- `src/models/experiments/svm_classifier_benchmark/experiment.py`: each
  SVM candidate config now runs inside its own `mlflow.start_run(...)`,
  logging hyperparameters and evaluation metrics per run for side-by-side
  comparison in the DagsHub UI

**Calibration Experiment**
- `src/models/experiments/winner_model_calibration/experiment.py`: logs
  base model params, per-method metrics (ROC AUC, F1, Brier score, log
  loss, reliability MAE) across all 18 calibration variants, tags the
  recommended method, and logs the summary/methods artifacts

**Airflow Integration**
- `docker-compose.yml`: airflow service now receives MLflow tracking
  credentials as environment variables (sourced from `.env`)
- `docker/airflow/airflow_requirements.txt`: added `mlflow-skinny==3.14.0`
  so the `train_winner_model` task can log to MLflow/DagsHub from inside
  the container
- Verified: end-to-end DAG run produces a new MLflow run + registry
  version on DagsHub

**DVC**
- Confirmed `dvc status` clean and all preprocessed datasets synced to
  the DagsHub DVC remote

### Test plan

- [x] `python -m src.models.train_winner` — run logged, model registered (v4)
- [x] `python -m src.models.experiments.svm_classifier_benchmark.experiment` — 4 comparison runs logged
- [x] `python -m src.models.experiments.winner_model_calibration.experiment` — run logged with calibration metrics
- [x] Airflow DAG (`end_to_end_mlops_pipeline`) executed end-to-end via docker-compose, including `train_winner_model` task
- [x] `dvc status` / `dvc push` clean, all data synced to DagsHub remote
- [x] `git status` clean on `feature/mlflow-tracking` before opening PR
