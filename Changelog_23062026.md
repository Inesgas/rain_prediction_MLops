## Changes — MLflow Integration & Bug Fixes (23/06/2026)

### Summary
- Added MLflow tracking to `src/models/train_winner.py` to log parameters, metrics, and model artifacts to DagsHub for each training run.
- Fixed the `locations_metadata.csv` path in `src/models/experiments/daily_zonal_baseline/experiment.py` (`data/processed/` → `data/preprocessed/`).
- Replaced `mlflow` with `mlflow-skinny` in `requirements.txt` to avoid the `pandas < 3` dependency conflict.
- Added `.env.example` as a template for local credentials.

### Notes
- `mlflow-skinny` provides the tracking and registry features required by this project without the full MLflow dependency stack.
- `dvc pull` and `dvc push` are the expected workflow for restoring and synchronizing DVC-tracked data on DagsHub.

### Next steps
1. Pull the latest branch changes.
2. Reinstall dependencies with `pip install -r requirements.txt`.
3. Copy `.env.example` to `.env` and add your DagsHub credentials.
4. Run `dvc pull`; if remote data is still missing, notify the team.
